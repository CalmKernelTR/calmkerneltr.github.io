Edition Methodology: BigFive → Fluent → Zenith

February 3, 2026 • BigFive Updater • Ahmet TANRIKULU

Starting with BigFive Updater v6.1.0, we adopted a new Edition system. In this post, I explain the rationale behind the system and how it works.

Why an Edition System?

As the project grew, version numbers alone (v5.0.0, v6.0.0) were no longer expressive enough. Each major version represents its own "era" — with different focus areas and different features.

Goal: Organize major versions into meaningful groups, giving each era a name that reflects its character.

Edition Table

Major Edition Codenames Focus
v5.x BigFive Alpine, Beacon, Dream 5 distro support, core foundation
v6.x Fluent ✓ Echo, Foxtrot Smooth UX, diagnostics, CI/CD
v7.x Zenith TBD Peak features (planned)

How Does It Work?

Edition = Major Version Series

Each major version (v5, v6, v7) has its own Edition. Edition names reflect the character of that era:

  • BigFive: Support for 5 major package managers (APT, DNF, Pacman, Zypper, APK)
  • Fluent: Smooth updates, seamless UX
  • Zenith: The peak, most advanced features

Codename = Minor/Patch Release

Releases within each Edition receive alphabetical codenames:

  • v6.0.x - v6.1.x = Echo (current)
  • v6.2.x = Foxtrot (next)

Format

"Edition Edition - Codename"

Example: "Fluent Edition - Echo" (v6.1.0)

What Changed in Practice?

CHANGELOG, ROADMAP, and README files now include Edition information. The script defines an EDITION="Fluent" variable.

Additionally, we created a Claude Code skill called codename-watcher. This skill checks Edition/Codename consistency across documentation — accidentally writing "BigFive Edition - Foxtrot" is no longer possible.

Future Plans

v6.2.0 "Foxtrot" is currently in development. Planned features include:

  • Hierarchical output (parent → container view)
  • Config file support (~/.config/bigfive/config)
  • More diagnostics

There are no concrete plans for v7.x "Zenith Edition" yet, but it will likely include major features such as a plugin system and GUI integration.

Conclusion

The Edition system is not just marketing — it makes tracking the project's evolution easier. You can immediately understand which version you are on and what the focus areas are for that era.
